## Lição #1 — 2026-05-15

**Você pediu:**
> "Na página pessoal em Contatos temos 3 botões: card, lista e Kanban.
>  Quero substituir o Kanban por um sistema de Temperatura de Contatos —
>  classificar contatos como Quente/VIP/Morno/Lead/Frio/Inativo com cores,
>  para saber quais clientes compram sempre, de vez em quando, ou ainda não fecharam negócio."

**Forma profissional:**
> "Substituir a view Kanban do módulo Contatos por um sistema de Temperatura —
>  6 status com badges coloridos (VIP, Quente, Morno, Lead, Frio, Inativo)
>  usando os tokens do design system OBSIDIAN. Temperatura persistida no localStorage.
>  View responsiva com scroll horizontal no mobile."

**Termos aprendidos:**
- **view** → forma como os dados são exibidos na tela (card, lista, Kanban, Temperatura)
- **módulo** → seção/página independente do app (Contatos, Finanças, Dashboard...)
- **badge** → etiquetinha colorida com um label (ex: "VIP", "Quente")
- **token** → variável CSS do design system (ex: `--signature` = cor ouro)
- **design system** → manual de identidade visual do app (cores, fontes, tamanhos)
- **localStorage** → onde o app salva dados no navegador, sem internet/servidor
- **view Kanban** → view com colunas que representam etapas de um fluxo
- **coluna** → divisão vertical numa tela (cada status = uma coluna na view Temperatura)

**Template usado:**
`"Quero substituir [o quê] por [o quê novo] no módulo [nome] — [detalhe do comportamento]"`

## Lição #6 — 2026-05-16

**Você pediu:**
> "quando eu fecho o menu lateral no computador ele só fecha e eu não tenho a opção de expandir ele novamente / e quando ele fecha os icones do menu lateral fica tudo estranho e aparecendo / quando ele fechasse poderíamos ter uma experiência melhor de visualização / e fora que falta algo para eu conseguir abrir o menu lateral no computador novamente / no mobile está normal e só no modo desktop que não está / mas se quiser revisar os dois modos e melhorar a experiência dos dois modos / enriqueça essa ideia"

**Forma profissional:**
> "Bugfix da sidebar no desktop: toggle desaparece ao colapsar (sem como reabrir), ícones corrompidos no rail mode. Causa raiz: botão de toggle estava dentro do container com overflow:hidden. Fix: introduzir wrapper .sidebar-inner que recebe o overflow, mover o toggle para fora dele. Enriquecer rail mode no desktop e drawer no mobile."

**Termos aprendidos:**
- **overflow: hidden** → Propriedade CSS que esconde tudo que ultrapassa a borda do container. O bug acontecia porque o botão de abrir/fechar ficava dentro de um elemento com essa propriedade — ele simplesmente sumia quando o menu fechava.
- **wrapper** → Um elemento HTML "pai" que envolve outros elementos só para organizar ou aplicar um estilo específico. No fix da sidebar, o `.sidebar-inner` é um wrapper: agrupa o conteúdo interno e recebe o overflow:hidden, enquanto o botão fica fora dele.
- **causa raiz** → O problema de verdade que causou o bug — não o sintoma. O sintoma era "botão sumindo"; a causa raiz era "botão dentro de container com overflow:hidden".

**Template para este tipo:**
`"Tem um bug no [componente]: [descreva o sintoma] no [desktop/mobile/ambos]. Enriqueça a experiência dos dois modos."`

## Lição #7 — 2026-05-16

**Você pediu:**
> "01 tínhamos programado a ter um dashboard dedicado ao pessoal e outro dedicado ao negócio oq vc acha disso / enriqueça essa ideia / 02 uma observação importante quando eu clicar no menu lateral em pessoal a tela do aplicativo deve ir para o dashboard pessoal e tudo mais mudo para o ambiente pessoal / quando eu clicar no menu negócio todo ambiente deve ir para meu Negócio e já deve ir para o dashboard do Negócio / 03 acho que ainda não temos o dashboard do meu negócio se não tiver crie uma experiência incrível / 04 quando eu estou em Pessoal a paleta de cores continua a padrão obsidian etc / quando eu Clico em negócio tudo tem q ter aquele toque corporativo / temos que fazer tudo isso c excelência e profissionalismo"

**Forma profissional:**
> "Implementar 4 melhorias integradas — Sprint Alpha (routing + paleta) e Sprint Beta (dashboards): (1) Dashboard 'Hub de Vida' (Pessoal) e 'Painel Executivo' (Negócio) com dados reais do localStorage; (2) Routing automático ao trocar de modo com fade transition 400ms e memória de posição por modo; (3) Toast de boas-vindas ao trocar de ambiente; (4) Paleta corporativa safira (--info) permeando todo o ambiente Negócio via CSS override — botões, foco, progresso, scrollbar, gradientes, superfícies."

**Termos aprendidos:**
- **toast** → Notificação temporária que aparece sozinha na tela e some após alguns segundos — sem precisar fechar manualmente. O Mentor24h exibe um toast ao trocar de modo: "Bem-vindo ao Modo Pessoal, Léo!" em ouro, "Modo Negócio ativado" em safira.
- **fade / cross-fade** → Efeito de transição onde uma tela "apaga" suavemente e outra "acende". No Mentor24h: 160ms para apagar o ambiente atual + 240ms para acender o novo = 400ms total. Parece uma troca de cena no cinema.
- **debounce** → Técnica que atrasa a execução de uma ação até o usuário parar de agir por X milissegundos. Exemplo: nota rápida com debounce de 800ms — só salva no localStorage 800ms depois que você parar de digitar. Evita salvar a cada letra digitada.
- **widget** → Componente visual independente que exibe um tipo de dado. No Dashboard Pessoal: widget de Agenda, widget de Saúde, widget de Finanças, widget de Contatos — cada um lê seus próprios dados e se atualiza de forma independente.
- **memória de posição** → O app lembra em qual módulo você estava antes de trocar de modo. Ao voltar para o Modo Pessoal, ele te leva de volta onde você estava — não força ir para o dashboard toda vez.

**Template para este tipo:**
`"Quero criar dois ambientes separados — [Pessoal] e [Negócio] — cada um com seu próprio dashboard, paleta de cores e routing automático ao trocar de modo. Ao clicar em [modo], o app vai direto para o dashboard daquele ambiente."`
